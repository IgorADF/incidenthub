# Creating a Domain Entity in IncidentHub

This project uses Clean Architecture with domain entities that manage their own identity and creation time. Every entity lives in `core/src/entities/` and mirrors a Prisma model in `core/src/db/schema.prisma`.

## Steps

1. **Add the Prisma model** to `core/src/db/schema.prisma`.
   - Use `@id` on the primary key, but **do not** use `@default(uuid())`.
   - Use `createdAt DateTime` but **do not** use `@default(now())`.
   - These values are generated by the entity class, not the database.
   - Map fields with `@map("snake_case")` and the table with `@@map("table_name")`.

2. **Create the entity class** at `core/src/entities/<name>.ts`.
   - Extend `DefaultEntity<T>` from `./_default`.
   - Define a frozen `interface` for the entity props (include `id` and `createdAt`).
   - Provide three static methods:
     - `create(props)` — accepts `Omit<IEntity, "id" | "createdAt">`; assigns `uuidv7()` id and `new Date()` createdAt.
     - `fromEntityToPrisma(entity)` — maps the entity to the generated Prisma payload type.
     - `fromPrismaToEntity(prismaEntity)` — maps a Prisma record back to the entity.

3. **Use the right imports.**
   - `import { DefaultEntity } from "./_default";`
   - `import { Prisma } from "../db/generated/client";`
   - `import { uuidv7 } from "uuidv7";`

## Template

```ts
import { DefaultEntity } from "./_default";
import { Prisma } from "../db/generated/client";
import { uuidv7 } from "uuidv7";

interface IExample {
  id: string;
  organizationId: string;
  name: string;
  createdAt: Date;
}

export class Example extends DefaultEntity<IExample> {
  static create(props: Omit<IExample, "id" | "createdAt">) {
    return new Example({
      id: uuidv7(),
      organizationId: props.organizationId,
      name: props.name,
      createdAt: new Date(),
    });
  }

  static fromEntityToPrisma(entity: Example): Prisma.ExampleGetPayload<object> {
    return {
      id: entity.getProps().id,
      organizationId: entity.getProps().organizationId,
      name: entity.getProps().name,
      createdAt: entity.getProps().createdAt,
    };
  }

  static fromPrismaToEntity(
    prismaEntity: Prisma.ExampleGetPayload<object>,
  ): Example {
    return new Example({
      id: prismaEntity.id,
      organizationId: prismaEntity.organizationId,
      name: prismaEntity.name,
      createdAt: prismaEntity.createdAt,
    });
  }
}
```

## Special cases

- **Passwords** should be typed as `HashedPassword` from `../types/hashed-password` and hashed outside the entity (use `hashPassword` from `../utils/password`).
- **Optional fields** in Prisma become `field: Type | null` in the entity interface; never use `undefined` in the interface.
- **Enums** from Prisma are cast to string unions inside the entity, e.g. `type: "ADMIN" | "DEV"`.
- **Soft-delete fields** such as `deletedAt` should be carried in the Prisma mapping but are not part of the core domain interface unless required by business rules.

## Verification

After creating the entity, run `npx tsc --noEmit` in `core/` and resolve any type errors against the generated Prisma client.
