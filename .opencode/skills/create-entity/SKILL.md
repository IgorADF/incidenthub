# Creating a Domain Entity in IncidentHub

This project uses Clean Architecture with domain entities that manage their own identity and creation time. Every entity lives in `backend/src/domain/entities/` and mirrors a Prisma model in `backend/src/infra/db/schema.prisma`.

## Steps

1. **Add the Prisma model** to `backend/src/infra/db/schema.prisma`.
   - Use `@id` on the primary key, but **do not** use `@default(uuid())`.
   - Use `createdAt DateTime` but **do not** use `@default(now())`.
   - These values are generated by value-objects in the entity class, not the database.
   - Map fields with `@map("snake_case")` and the table with `@@map("table_name")`.

2. **Create value-objects** if the entity needs new identifiers or dates.
   - Existing value-objects live in `backend/src/domain/value-objects/`.
   - Prefer `readonly value` fields.
   - Validate on construction when possible (e.g. UUIDv7 format).
   - Provide `equals(other)` for comparisons.

3. **Create the entity class** at `backend/src/domain/entities/<name>.ts`.
   - Extend `DefaultEntity<T>` from `./_default`.
   - Define a frozen `interface` for the entity props.
   - Use `UUIDv7` for `id`, `CreatedAt` for `createdAt`, and `AssociationUUIDv7` for foreign keys.
   - Provide a single static method:
     - `create(props)` — accepts `Omit<IEntity, "id" | "createdAt">`; assigns `new UUIDv7()` and `new CreatedAt()`.
   - Do **not** add Prisma conversion methods to the entity.

4. **Create the mapper** at `backend/src/infra/mappers/<name>.ts`.
   - Provide `fromEntityToPrisma(entity)` and `fromPrismaToEntity(raw)`.
   - The mapper may use `new Entity(...)` to reconstruct persisted data.

## Template

```ts
// backend/src/domain/entities/example.ts
import { DefaultEntity } from "./_default";
import { UUIDv7 } from "@domain/value-objects/uuidv7";
import { CreatedAt } from "@domain/value-objects/created-at";

interface IExample {
  id: UUIDv7;
  name: string;
  createdAt: CreatedAt;
}

export class Example extends DefaultEntity<IExample> {
  static create(props: Omit<IExample, "id" | "createdAt">) {
    return new Example({
      id: new UUIDv7(),
      name: props.name,
      createdAt: new CreatedAt(),
    });
  }
}
```

```ts
// backend/src/infra/mappers/example.ts
import { Example } from "@domain/entities/example";
import { Prisma } from "@infra/db/generated/client";
import { UUIDv7 } from "@domain/value-objects/uuidv7";
import { CreatedAt } from "@domain/value-objects/created-at";

export class ExampleMapper {
  static fromEntityToPrisma(
    entity: Example,
  ): Prisma.ExampleGetPayload<object> {
    return {
      id: entity.getProps().id.value,
      name: entity.getProps().name,
      createdAt: entity.getProps().createdAt.value,
    };
  }

  static fromPrismaToEntity(
    prismaEntity: Prisma.ExampleGetPayload<object>,
  ): Example {
    return new Example({
      id: new UUIDv7(prismaEntity.id),
      name: prismaEntity.name,
      createdAt: new CreatedAt(prismaEntity.createdAt),
    });
  }
}
```

## Special cases

- **Passwords** should be typed as `HashedPassword` from `~types/hashed-password` and hashed outside the entity (use `hashPassword` from `@utils/password`).
- **Optional fields** in Prisma become `field: Type | null` in the entity interface; never use `undefined` in the interface.
- **Enums** from Prisma are cast to string unions inside the entity, e.g. `type: "ADMIN" | "DEV"`.
- **Soft-delete fields** such as `deletedAt` should be carried in the mapper but are not part of the core domain interface unless required by business rules.

## Verification

After creating the entity and mapper, run `npx tsc --noEmit` in `backend/` and resolve any type errors against the generated Prisma client.
