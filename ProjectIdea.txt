FLOW Worker com HealthCheck e Incident

Service
   Campos:
        status: z.string().min(1).max(50), -> "CHECKING", "INCIDENT", "DISABLED"
        url: URL,
        intervalSeconds: z.number().int().positive().min(5).max(99999), -> Checking time from interval
        timeoutSeconds: z.number().int().positive().min(5).max(20), -> Request timeout 
        expectedResponseStatus: z.number().int().min(100).max(599), -> 
        incidentDetectionFails: z.number().int().positive().max(99), -> QTD to incident seguidos
        consecutivesIncidentDetectionFails: z.number().int().min(0).max(99999), ->
        emailToAlert: Email.nullable(), ->
        enabled: z.boolean(), -> not run if == false
        createdAt: CreatedAt,
    Duvidas:
        -> Adiciona na queue sempre que adicionar um servico ou atulizar ele? Ou sempre que o worker rodar sera pego do banco?
            -> atualiza dados da fila sempre que atualizar ele? (exeto para de executar ele - enabled = true)
            -> remove da fila sempre que deleta um servico?
        -> Precisa dar algum lock via db ou logico no registro?
        -> Funcao para repopular a fila com servicos caso precise (select service == enabled)
        
Flow:
- Pesquisou registro - Ou pega de algum cache/objeto da fila
- Se estiver desabilitado o servico, pula 
- Call api pela URL e com um timeout esperado
    - se dar timeout, cria um healthcheck apontando timeout e soma na contagem de incident
    - se retornar um status errado, cria um healthcheck apontando erro e soma na contagem de incident
    - se retornar status certo, cria um healthcheck com sucesso
